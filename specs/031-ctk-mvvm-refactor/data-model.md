# Data Model: CommunityToolkit.Mvvm ViewModel Refactor

**Date**: 2026-05-31
**Feature**: CommunityToolkit.Mvvm ViewModel Refactor

## Overview

This feature does not introduce new persistent data entities. Instead, it restructures the in-memory presentation layer (ViewModels) to use compile-time source generation. The "data model" documented here describes the structural changes to ViewModel classes, their generated artifacts, and the relationships between them.

## Entity: ViewModel (Refactored)

**Represents**: The presentation logic for a specific UI screen, widget, or shell component.

### Attributes (before refactor)

| Attribute | Type | Description |
|-----------|------|-------------|
| Manual properties | `private T _field; public T Property { get { return _field; } set { _field = value; OnPropertyChanged("Property"); } }` | Hand-written INPC boilerplate |
| Commands | `ICommand` properties backed by `RelayCommand` or `AsyncRelayCommand` instances | Manual command instantiation |
| Base class | `ViewModelBase` implementing `INotifyPropertyChanged` manually | Custom base with manual `SetProperty<T>` helper |

### Attributes (after refactor)

| Attribute | Type | Description |
|-----------|------|-------------|
| Source-generated properties | `[ObservableProperty] private T _field;` | Compile-time generation of `Property` with INPC |
| Source-generated commands | `[RelayCommand] private void ExecuteAction()` or `async Task ExecuteActionAsync()` | Compile-time generation of `ActionCommand` with `ICommand` |
| Base class | `ViewModelBase : ObservableObject` | Inherits from `CommunityToolkit.Mvvm.ComponentModel.ObservableObject` |
| Partial method hooks | `partial void OnPropertyNameChanged(T value)` | Hook for side effects previously in custom setters |

## Entity: Generated Binding Code

**Represents**: Compile-time artifacts produced by the CommunityToolkit.Mvvm source generator.

### Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| Generated file | `*.g.cs` | Auto-generated partial class file in `obj/` |
| Generated property | `public T Property { get; set; }` with `SetProperty(ref _field, value)` | Observable property with change notification |
| Generated command | `public IRelayCommand ActionCommand { get; }` | Command wrapping the annotated method |
| Command parameter | `IRelayCommand<T>` or `IAsyncRelayCommand<T>` | Generic variant when method accepts a parameter |

## Relationships

```
ViewModelBase (ObservableObject)
    |
    +-- HomeViewModel (partial)
    +-- SettingsViewModel (partial)
    +-- BOQActivityGeneratorViewModel (partial)
    +-- DurationEstimatorPageViewModel (partial)
    +-- WBSEditorViewModel (partial)
    +-- WBSGeneratorViewModel (partial)
    +-- RelationshipGeneratorViewModel (partial)
    +-- Dashboard/*WidgetViewModel (partial)
    +-- Primavera/*ViewModel (partial)
    +-- DiagnosticsViewModel (partial)
    +-- LanguagePageViewModel (partial)
    +-- ShellViewModel (partial)
    +-- CommandPaletteViewModel (partial)
    +-- ToastViewModel (partial)
    +-- ... (remaining ViewModels)
```

**Relationship**: Each concrete ViewModel inherits from `ViewModelBase` (which inherits `ObservableObject`). Each concrete ViewModel is declared as `partial class` to allow the source generator to augment it with generated properties and commands in a companion `*.g.cs` file.

## State Transitions

| State | Description | Entry Condition |
|-------|-------------|-----------------|
| Manual | ViewModel uses hand-written INPC and custom RelayCommand | Initial state |
| Migrated-Partial | ViewModel declared `partial`; properties/commands annotated with `[ObservableProperty]` / `[RelayCommand]` | After batch migration |
| Verified | Build succeeds; `*.g.cs` artifacts present; smoke test passes | After build validation |
| Clean | Obsolete `RelayCommand.cs` and `AsyncRelayCommand.cs` deleted; verification script passes | Final state |

## Validation Rules

- **VR-001**: Every migrated ViewModel MUST be declared as `partial class`.
- **VR-002**: No manual `OnPropertyChanged(string)` calls MUST remain inside property setters.
- **VR-003**: No references to `Models.RelayCommand` or `Helpers.AsyncRelayCommand` MUST remain.
- **VR-004**: All `ICommand`-typed properties MUST be generated by `[RelayCommand]` (not manually instantiated).
- **VR-005**: `ViewModelBase` MUST inherit `CommunityToolkit.Mvvm.ComponentModel.ObservableObject`.
